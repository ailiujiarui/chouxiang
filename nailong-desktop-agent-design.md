# Nailong 桌宠与代码审判助手集成设计

日期：2026-07-19
状态：第二阶段目标设计，依赖代码审判核心完成

## 产品定位

Nailong 是具有完整桌宠属性的常驻桌面角色，也是代码审判助手的桌面交互入口。它不是独立于分析核心的旁支，也不能退化成托盘通知器、普通聊天窗口或 Dashboard 浮层。

代码审判引擎负责 AST、多 Agent、验证、裁决和完整报告；桌宠负责陪伴、用户手势、进度演出、短判词和打开完整工作台。

## 桌宠本体

### 可见形态

- 透明背景、置顶、无边框、可拖拽；
- 多显示器和 DPI 适配；
- 待机、观察、思考、吐槽、得意、困倦、庆祝、担忧状态；
- 每个状态对应动画 clip、表情、停留时间和可中断规则；
- 对话气泡、点击反馈、拖放高亮和任务进度演出。

### 本地互动

- 单击、双击、拖拽和右键菜单；
- 托盘暂停、安静模式、退出和清除本地历史；
- 冷却、重复消息去重和每小时主动打扰预算；
- 会议、演示、全屏游戏/视频时默认不主动弹窗；
- 分析服务不可用时仍能待机、响应点击和给出固定人格回复。

## 分析入口

用户可以通过以下显式动作请求代码审判：

- 把 `.py` 文件拖到桌宠；
- 打开代码粘贴面板；
- 选择本地文件；
- 输入 canonical GitHub 仓库 URL；
- 从受支持 IDE 插件显式发送当前文件或选区。

桌宠不得根据前台窗口或活动识别自动读取、提交或上传代码。

## 分析演出

| 分析阶段 | 桌宠状态 | 展示 |
| --- | --- | --- |
| 输入校验 | 观察 | 检查文件/URL |
| AST Analyst | 观察/思考 | 热点和复杂度提示 |
| Minimizer | 思考 | “正在压缩分支” |
| Defender | 担忧 | 行为保持风险 |
| Adversary | 紧张/挑衅 | 反例与攻击计数 |
| Judge | 等待裁决 | 证据等级 |
| 通过 | 得意/庆祝 | 短判词和指标 |
| 不充分或失败 | 担忧/吐槽 | 风险与下一步 |

完整源码、diff、日志和报告不塞进气泡；用户点击结果后打开工作台对应 task。

## 共享事件

```text
AnalysisRequested
AnalysisProgressed
AnalysisCompleted
AnalysisFailed
PetStateChanged
PopupDecision
```

事件只携带 task ID、阶段、进度、证据等级、指标摘要和脱敏短文案。完整代码通过分析 API 的受控输入传递，不经过桌宠 EventBus 广播。

## 人格

桌宠人格与报告人格共享版本化 persona 定义，但表现层不同：

- 报告人格负责长篇技术审判；
- 桌宠人格负责短句、动作、表情和时机；
- 桌宠可以傲娇、毒舌和得意，但不能羞辱用户身份、外貌、能力或群体；
- 技术失败时不能用人格话术掩盖证据不足。

短期分析结果可以推动情绪状态，状态超时后必须回到桌宠自己的基线人格，而不是永久停留在任务状态。

## 隐私与打扰

- 默认不采集截图、OCR、剪贴板、完整窗口标题、终端正文或源代码；
- 活动采集只产生应用类别、空闲时长、会议/全屏等有限信号；
- `private` 和 `blocked` 事件不进入模型；
- 用户可以暂停采集、暂停弹窗和清除全部本地历史；
- 分析请求和活动陪伴是两条独立授权路径。

## 当前骨架

现有代码仅提供 Pydantic 事件、EventBus、单实例锁、Null renderer 和最小 PySide6 window。它不是完成的桌宠，也尚未接入分析核心。

现有 `ActivityEvent`、`PopupDecision` 和 EventBus 可以保留；必须新增 PetState、分析事件、动画资源协议、交互状态机和分析 API bridge。

## 验收

- 不启动分析服务也能独立运行和互动；
- 具有可辨识的桌宠动画、状态、拖拽、点击、气泡和托盘控制；
- 文件/代码/URL 可以显式发起同一分析核心任务；
- 分析进度准确映射为桌宠演出；
- 短判词与 Dashboard 完整报告的裁决和证据等级一致；
- 会议、全屏、安静模式、冷却和隐私边界没有绕过路径；
- 完整 code review 后再申请提交、推送或部署。
