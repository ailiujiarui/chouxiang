# Nailong 配置与持久化任务

日期：2026-07-24  
状态：已确认，待实施

## 目标

为 Nailong 桌面进程建立独立、可验证且跨重启生效的配置与持久化能力。实现必须延续现有“隐私数据与通知数据分离”的边界，并为后续活动采集、识别器、人格决策和渲染层提供稳定接口。

本任务只负责桌宠的配置、SQLite 数据模型、迁移、启动接入与测试。活动采集器的最小 Windows 前台信号实现已在后续阶段接入；状态分类器、DeepSeek 桌宠客户端、人格图和动画渲染仍不在本任务范围内。

## 现状

- `refactor_agent.config.AppSettings` 已服务于代码审判主进程，不承载桌宠设置。
- `PrivacyStore` 已保存首次授权和最小化活动事件；`NotificationStore` 已保存通知队列、游标、免打扰开关和 ACK。
- 通知数据库默认位于 `.runs/nailong_notifications.sqlite`，隐私数据库从锁文件目录推导，路径策略不一致。
- 普通通知已有 5 至 15 分钟冷却，手动免打扰已经持久化；每日最大弹窗次数、免打扰时段、人格强度和用户应用规则尚未实现。

## 配置模型

新增 `nailong_agent.config.NailongSettings`，与 `AppSettings` 保持独立。它负责解析桌宠运行所需的不可变默认值和命令行临时覆盖项。

有效值按以下优先级确定：

```text
CLI 临时参数 > 已持久化的用户偏好 > 环境变量默认值 > 代码默认值
```

其中，CLI 参数只影响本次运行；用户在桌宠界面修改的偏好必须写入 SQLite，并在后续启动时恢复。环境变量作为首次启动或尚未保存偏好时的默认值。

### 设置字段

| 字段 | 用途 | 保存位置 |
| --- | --- | --- |
| `data_dir` | 隐私库、通知库和锁文件的统一根目录 | CLI/环境变量/代码默认值 |
| `analysis_url` | 订阅代码审判分析事件的服务地址 | CLI/环境变量/代码默认值 |
| `activity_listener_enabled` | 是否允许已授权的活动监听器运行 | 用户偏好 |
| `manual_pause_enabled` | 是否暂停桌宠主动监听和主动弹窗 | 用户偏好 |
| `do_not_disturb_start`、`do_not_disturb_end` | 每日免打扰时间段，支持跨午夜 | 用户偏好 |
| `minimum_cooldown_seconds`、`maximum_cooldown_seconds` | 普通通知随机冷却范围 | 用户偏好 |
| `maximum_popups_per_day` | 每个本地自然日允许开始显示的最大通知数 | 用户偏好 |
| `personality_intensity` | `LOW`、`STANDARD` 或 `HIGH` | 用户偏好 |
| `deepseek_model` | 桌宠专用推理客户端未来使用的模型名 | CLI/环境变量/代码默认值 |
| 应用白名单/黑名单 | 允许或抑制监听的应用类别规则 | 用户偏好 |

DeepSeek API Key 只允许从进程环境读取，禁止进入设置对象的序列化输出、日志、运行产物和 SQLite。

### 路径约定

`data_dir` 默认值为 `.runs`。桌面进程从它派生以下路径：

```text
<data_dir>/nailong-agent.lock
<data_dir>/nailong_privacy.sqlite
<data_dir>/nailong_notifications.sqlite
```

`DesktopProcess`、CLI 和 `scripts/start.ps1` 必须使用同一套路径解析逻辑。调用方仍可为测试或诊断显式传入数据库路径。

## 数据库职责

继续使用两个 SQLite 文件，禁止将两类数据合并，也不写入代码审判主库 `refactor_agent.sqlite`。

### 隐私数据库

隐私库只保存经过 `PrivacyPolicy` 最小化且为 `public` 的桌面活动信息。

| 表 | 职责 |
| --- | --- |
| `pet_privacy_consent` | 首次授权和本地/远程推理授权状态 |
| `pet_activity_events` | 去重后的最小化活动事件及稳定指纹 |
| `pet_activity_windows` | 按应用类别、活动类型和时间窗口聚合的统计记录 |

活动事件只能包含事件 ID、时间、来源、归一化应用类别、活动类型、置信度和允许的最小化元数据。不得保存窗口标题、文件路径、原始代码、终端正文、剪贴板、截图、OCR 文本、令牌、密码或其他密钥。

“删除本地活动记录”只删除 `pet_activity_events` 与 `pet_activity_windows`，不得删除已记录的授权选择。

### 通知数据库

通知库保存与打扰策略和表现状态有关的信息，不存桌面原始内容。

| 表 | 职责 |
| --- | --- |
| 既有 `notification_*` 表 | 消费游标、通知意图、通知状态、终态抑制和运行时状态 |
| `pet_preferences` | 单行桌宠偏好，包括暂停、免打扰时段、冷却、每日预算和人格强度 |
| `pet_app_rules` | 归一化应用类别的白名单和黑名单 |
| `pet_personality_state` | 当前情绪、关联任务、更新时间和过期时间 |
| `notification_daily_budget` | 按本地日期统计已经开始显示的弹窗次数 |

人格状态到期后必须恢复为默认待机状态，不能在重启或旧任务完成后无限期保留。

## 规则与优先级

每次活动监听、活动入库、通知入队或通知显示前，按下列顺序执行决策：

```text
隐私授权和敏感内容拦截
  > 手动暂停
  > 会议、演示和全屏抑制
  > 免打扰时段或手动免打扰
  > 每日弹窗预算
  > 普通通知冷却
  > 人格表达与渲染
```

- 敏感内容、会议、`private` 与 `blocked` 事件不得持久化为活动历史、不得调用远程推理，也不得生成主动人格化内容。
- 手动暂停和免打扰可阻止主动弹窗；它们不得被人格强度、优先级或 DeepSeek 结果绕过。
- 每日预算以本地自然日计数，计数时机为通知进入 `DISPLAYING` 状态而不是仅入队。
- 默认情况下，终态通知也受每日预算限制。未来若提供终态例外开关，必须是明确、可见且可测试的配置，仍不得绕过手动暂停和免打扰。
- 人格强度只影响文案与动作选择，不得改变分类结果、隐私判断、规则优先级或弹窗次数。

## 迁移与可靠性

两个数据库分别维护迁移版本。迁移实现可以沿用现有表结构探测方式，或使用明确的 schema version 表；同一数据库内必须始终选用一种可测试的方法。

- 初始化与升级迁移必须幂等，重复启动不得重复建表、重复改列或丢失数据。
- 每个迁移在事务内完成；失败时保留原始数据库，不得以删除数据库作为恢复方式。
- 迁移必须兼容现有已创建的 `nailong_privacy.sqlite` 与 `nailong_notifications.sqlite`。
- SQLite 连接策略应统一启用外键检查和合理的 busy timeout；并发写入失败不得导致扩大采集、放宽隐私规则或重复发送通知。
- 配置无效时，桌宠应拒绝启动相应监听能力并报告不含敏感值的错误；缺失可选配置应回退到安全默认值。

## 实施任务

1. 创建 `src/nailong_agent/config.py`，定义 `NailongSettings`、路径解析、环境变量解析和 CLI 覆盖接口。
2. 调整 `src/nailong_agent/app.py`，通过 `NailongSettings` 创建锁、隐私库、通知库和分析事件订阅器，并保留依赖注入测试入口。
3. 调整 `scripts/start.ps1`，增加桌宠数据目录和配置覆盖参数，将它们传递给独立桌面进程。
4. 扩展 `src/nailong_agent/privacy_store.py`，实现最小化活动事件的稳定去重、时间窗口聚合、迁移和定向清除。
5. 扩展 `src/nailong_agent/notification_store.py`，实现用户偏好、应用规则、人格短期状态、每日预算和迁移。
6. 调整 `NotificationService` 与通知策略，使其读取有效偏好并落实预算、暂停和免打扰时段。
7. 定义采集器、识别器和人格图所需的存储接口；后续阶段只实现最小前台信号采集，不实现窗口标题、IDE/终端正文、轻量分类、远程推理或动画资源。
8. 更新 README 或桌宠启动说明，记录新增环境变量、默认路径、数据保留规则和清除活动记录的影响范围。

## 验收标准

- CLI、已保存偏好、环境变量和默认值按既定优先级生效。
- 重启桌宠后，手动暂停、免打扰状态/时段、每日预算、人格强度和应用规则保持不变。
- 每日预算在本地日期切换后重新计数，并且不会因重启绕过。
- 跨午夜免打扰时段正确生效；会议、全屏、手动暂停和免打扰的优先级高于人格表现。
- 旧版隐私库和通知库可以原地升级，已有授权、通知游标和待处理通知不丢失。
- 相同活动事件不会重复写入；同类连续活动会按规定时间窗口聚合。
- 数据库中不存在窗口标题、源代码、剪贴板、截图、OCR、终端正文、令牌、密码或 API Key。
- “删除本地活动记录”不会删除授权、通知队列、免打扰偏好或代码审判运行记录。
- 新增测试至少覆盖：配置优先级、路径派生、旧库迁移、跨重启恢复、跨午夜规则、每日预算、活动去重/聚合和敏感数据拒绝写入。

## 活动采集实施记录

- Windows 前台采集器使用 `SetWinEventHook(EVENT_SYSTEM_FOREGROUND)`，在独立线程中仅解析前台进程 ID 与可执行文件名；不读取窗口标题。
- 采集器同时使用 `GetLastInputInfo` 每 15 秒采样系统空闲秒数；连续空闲达到 5 分钟仅发布一次 idle 事件，恢复活动后重置，并以窗口矩形与当前显示器矩形的比较生成全屏信号。
- 首次授权、持久化监听开关、手动暂停和应用白名单/黑名单均在事件持久化和发布前检查；同一应用五秒内的前台切换会在内存中节流。
- 最小化统一事件只会写入归一化应用类别、活动类型、置信度和固定格式摘要。空闲秒数、全屏和会议信号只在隐私边界内参与分类和拦截；窗口标题、路径、代码、终端内容、剪贴板、截图、OCR 与密钥没有采集或持久化路径。
- `DesktopProcess` 在授权结果确定且事件总线启动后启动采集器，并在事件总线停止前停止它。`NAILONG_ACTIVITY_LISTENER_ENABLED` 或 `--no-activity-listener` 仅决定是否创建采集器；持久化偏好仍是运行期门禁。
- Windows 之外使用空采集源，不收集任何桌面活动。

## 非目标

- 不实现窗口标题、IDE/终端读取、截图或 OCR。
- 不实现规则分类器、轻量分类器或桌宠专用 DeepSeek 客户端。
- 不实现 `PetGraphState`、人格推理节点或 Prompt 模板。
- 不实现 PNG/GIF/动画资源、互动窗口或新的渲染控件。
- 不修改 `refactor_agent` 的代码审判数据库、LLM 重构 Prompt 或 FastAPI 合同。
