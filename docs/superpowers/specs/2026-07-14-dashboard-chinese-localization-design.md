# 运维仪表盘中文化设计

日期：2026-07-14
状态：已实施、验证、自审并于 2026-07-15 获得集成许可

## 目标

将四页签 Streamlit 运维仪表盘的用户可见界面统一为简体中文，同时保持 API、数据库、状态机和控制接口的英文协议值不变。中文化只发生在展示层，不改变 Worker、Webhook、基准测试或重构流程的行为。

现有旧版“重构 Agent 竞技场”已经基本使用中文；本次重点覆盖 `dashboard_main()` 提供的运维界面，并修正两套界面中残留的不一致术语。

## 实现方案

采用展示层集中翻译方案：

- `dashboard.py` 负责页面标题、侧栏、页签、按钮、指标、选择器、空状态和操作反馈等中文文案。
- `dashboard_views.py` 负责表格列名、状态显示、Agent 名称、裁决结果和事件标签等可测试的纯数据转换。
- API 响应、SQLite 记录、状态判断和控制请求继续使用原始英文值。
- 不引入通用 i18n 框架，不增加语言切换器；当前产品界面固定使用简体中文。

该方案避免翻译逻辑散落在渲染代码中，也不会让中文文本进入协议层或影响现有状态机判断。

## 中文化范围

需要中文化的用户可见内容包括：

- 页面标题和浏览器标题；
- 侧栏字段、刷新按钮和记录数量限制；
- “任务”“执行过程”“代码变更”“基准测试”四个页签；
- 任务状态、尝试次数、租约所有者、截止时间、剩余时间和关联运行等指标名；
- 取消、重试、选择任务、选择运行和选择基准测试等控件；
- 空数据、鉴权失败、状态冲突、网络失败和数据加载失败提示；
- 任务、事件、执行轨迹和基准测试表格的列名；
- 已知状态、Agent、裁决和失败类别的展示文本。

以下内容保持原样：

- API URL、仓库 URL 和 PR URL；
- Job ID、Run ID、事件 ID 和 Worker ID；
- 文件名、qualified name、Git SHA、仓库名和模型名；
- Python 源码、统一 diff、pytest 日志、对抗日志和服务端返回的原始诊断正文；
- API 字段名、枚举值、数据库内容和 HTTP 请求格式。

## 状态与术语

已知状态采用“中文（原值）”格式，例如：

- `QUEUED` -> `排队中（QUEUED）`
- `RUNNING` -> `运行中（RUNNING）`
- `CANCEL_REQUESTED` -> `取消请求中（CANCEL_REQUESTED）`
- `CANCELLED` -> `已取消（CANCELLED）`
- `TIMED_OUT` -> `已超时（TIMED_OUT）`
- `SUCCESS` -> `成功（SUCCESS）`
- `FAILED` -> `失败（FAILED）`
- `DRY_RUN` -> `试运行完成（DRY_RUN）`

未知状态显示为 `未知状态（<原值>）`，不静默丢弃原始值。控制按钮是否可用仍由原始状态决定，禁止对中文文本做业务判断。

固定术语如下：

| 英文 | 中文 |
| --- | --- |
| Tasks | 任务 |
| Execution | 执行过程 |
| Code | 代码变更 |
| Benchmarks | 基准测试 |
| Job | 任务 |
| Run | 运行记录 |
| Lease owner | 租约所有者 |
| Deadline | 截止时间 |
| Remaining | 剩余时间 |
| Attempts | 尝试次数 |
| Cancel | 取消任务 |
| Retry | 重新执行 |
| Adversary | 对抗测试 |

`pytest`、AST、LOC、CC、Token、USD、API 和 Agent 等领域缩写保留，不进行生硬翻译。

## 错误处理

Dashboard API 客户端继续保留状态码和服务端详情。界面为常见类别增加中文上下文：

- `401`：管理员令牌无效或缺失；
- `404`：任务、运行记录或产物不存在；
- `409`：当前状态不允许执行该操作；
- 连接失败或超时：无法连接本地 API；
- 其他错误：显示中文操作说明，并附带经过现有脱敏处理的原始详情。

不得翻译或重写源代码、测试日志和服务端诊断内容，以免丢失排障信息。

## 数据与控制流

1. Dashboard API 客户端读取原始 JSON 数据。
2. `dashboard_views.py` 将原始记录转换为包含中文列名和展示值的视图数据。
3. `dashboard.py` 渲染中文控件和表格。
4. 用户触发取消或重试时，界面仍将原始 Job ID 和英文状态契约提交给 API。
5. 控制完成后重新读取服务端数据，不在 Streamlit 本地推测最终状态。

## 测试

采用测试驱动方式覆盖：

- 状态翻译、未知状态回退和原始值保留；
- 任务、事件、执行轨迹和基准测试表格的中文列名；
- 四个中文页签、侧栏字段、按钮和空状态；
- 取消与重试仍按英文状态正确启用或禁用；
- 常见 HTTP 错误显示中文上下文且保留脱敏详情；
- Admin Token 不出现在 URL、日志、表格或持久化数据中；
- Streamlit AppTest 和本地健康检查继续通过。

实施完成后运行 Dashboard 聚焦测试、完整 `pytest -q`、Streamlit 健康检查和页面人工检查，并对所有改动执行完整代码审查与自我修复。

## 非目标

- 不增加中英文切换；
- 不修改 FastAPI 返回内容或数据库字段；
- 不翻译用户仓库、Issue、源码、日志或模型输出；
- 不调整页面布局、权限模型、Worker 行为或 Docker 配置；
- 不提交、推送或部署本次改动，除非获得后续单独许可。

## 验收标准

- 运维仪表盘所有固定用户可见文案均为简体中文；
- 技术标识和诊断原文保持可复制、可追踪；
- 原始状态仍驱动控制逻辑，中文文本只负责展示；
- 旧竞技场与新运维仪表盘使用一致术语；
- Dashboard 聚焦测试、完整测试和运行时健康检查通过；
- 代码审查没有未解决的 Critical 或 High 问题。
